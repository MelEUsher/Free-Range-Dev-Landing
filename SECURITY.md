# Security Policy

## Reporting a Vulnerability

The Free Range Dev takes the security of this project seriously. If you
believe you've found a security vulnerability, please report it privately
so we can address it before it's publicly disclosed.

**Please do not open a public issue for security vulnerabilities.**

You can report a vulnerability through either of these channels:

- **GitHub Private Vulnerability Reporting** (preferred): Use the
  ["Report a vulnerability"](https://github.com/MelEUsher/Free-Range-Dev-Landing/security/advisories/new)
  button in the Security tab of this repository.
- **Email:** squad@thefreerangedev.dev

### What to include

To help us resolve the issue quickly, please include where possible:

- A description of the vulnerability and its potential impact
- Steps to reproduce, or a proof of concept
- The affected file(s), URL(s), or component(s)

### What to expect

- We aim to acknowledge reports within a few business days.
- We'll keep you informed as we investigate and work on a fix.
- Once resolved, we're happy to credit you for the responsible
  disclosure if you'd like.

Thank you for helping keep The Free Range Dev and its users safe.
